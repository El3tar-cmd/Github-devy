Architecture Overview (نظرة عامة على المعمارية)

The Devy platform acts as a secure, sandboxed web-based IDE designed to serve autonomous AI coding agents (like Gemini 2.5/3.5 and local Ollama models). The platform splits logic into modular routes and WebSocket controllers, running inside a local Termux/Linux node runtime.

01 / Modular Backend Router

Express controllers modularized cleanly under /server/routes/ to manage filesystem, shells, workspace listing, package installation, and database interactions.

02 / WebSocket Tunneling

Real-time terminal shell proxy mapped directly to system shells (bash/sh) allowing persistent bash command execution across tab transitions and UI refreshes.

03 / State separates (React Context)

Frontend uses structured contexts (WorkspaceContext & AgentContext) to decouple AI chat processes from workspace filesystem nodes.

Key Platform Features (أهم ميزات المنصة)

  • Token-Saving Summary Engine: Auto-compresses chat history exceeding 8 turns using LLM summarization, dropping older raw logs and injecting a clean summary context inside system prompts.
  • Interactive Plan Board: Checkable Todo checklist which loads .github-devy/tasks.md and writes toggled items directly back to the project files.
  • Port-Release Signal Shutdown: Custom process handlers on SIGINT and SIGTERM kill all spawned background processes and terminal sockets, fully releasing port 9876.

Port Configuration & Process Cleanups

Devy runs on an unusual non-standard port 9876. This isolates the platform server from active web-development projects (which typically lock standard ports like 3000, 5000, 5173, etc.).

// Clean Shutdown Implementation inside server.ts

process.on('SIGINT', () => handleShutdown('SIGINT'));

process.on('SIGTERM', () => handleShutdown('SIGTERM'));


function handleShutdown(signal) {

  cleanAllTerminalSessions(); // SIGKILL to all active bash shells

  killAllBackgroundProcesses(); // SIGKILL to all Vite/Node dev servers

  process.exit(0);

}

Active Plan Mode (نمط التخطيط النشط)

Plan Mode is designed to tackle complex coding milestones or initialize new codebases. When enabled, Devy is directed to create two roadmap files in the project workspace:

📋 .github-devy/plan.md

Contains the overall project specifications, architecture details, dependencies, and implementation notes in markdown formatting.

✅ .github-devy/tasks.md

Granular checklist tasks using markdown checkbox syntax (- [ ] and - [x]). Read and rendered interactively in the IDE Planner tab.
